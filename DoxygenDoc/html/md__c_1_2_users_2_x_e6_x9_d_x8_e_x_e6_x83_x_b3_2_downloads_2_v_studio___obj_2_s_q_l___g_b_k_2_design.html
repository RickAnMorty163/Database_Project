数据库关系模式设计

1.初始的关系模式 R1 员工(Employee),
(
  员工编号(emp_id),
  姓名(name),
  性别(sex),
  年龄(age),
  科室(department),
  职位(position)
)

R2 工资(Salary)
(
  工资编号(salary_id),
  员工编号(emp_id),
  基本工资(basic_salary),
  福利补贴(welfare_salary),
  奖励工资(reward_salary),
  扣除奖金(unemp_insurance),
  实发工资(actual_salary),
  发放日期(pay_date)
)

2.第三范式(3NF)优化
R1 Employee
  emp_id (Prime Key)
  name
  sex
  age
  position (Foreign Key reference by R3 PK)
  department (Foreign Key reference by R4 PK)
  basic_salary
  weflare_subsidy
  reward_salary
  unemp_insurance
  housing_fund

R2 SalaryRecord
  sal_id (Prime Key) -> auto increase id
  emp_id (Prime Key)
  actual_salary (rely to R1 attribute)
  pay_date

R3 Position_Tab
  pos_name (Prime Key) -> append concept to R1 (pos_name)

R4 Department_Tab
  dep_name (Prime Key) -> append concept to R1 (department)

R5 User
  user_name (Prime Key)
  password
